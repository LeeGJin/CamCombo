# Vibe Coding 과제 - 카메라 조합 비교 사이트 구현 계획

## Context
소프트웨어공학 과제로 Vibe Coding을 소개하는 PPT + Web Page 개발 예제를 제작.
- 주제: **카메라 바디 + 렌즈 조합 비교 사이트**
- 동기: 본인이 사진 촬영이 취미인데, 카메라/렌즈 조합을 한눈에 비교하기 어려운 불편함 해결
- 기술 스택: HTML, CSS, JavaScript만 사용 (프레임워크 X)
- 청중: 40~50대 직장 상사 (고가 장비 비교/구매에 공감 가능)
- 마감: 2026-03-20 오후 5시

## 구현 계획

### 파일 구조
```
SE/
├── PLAN.md       ← 이 파일
└── index.html    ← 단일 파일 (CSS, JS 내장)
```

### 페이지 구성

#### 1. 헤더
- 사이트 제목: "CamCombo - 카메라 조합 비교"
- 간단한 설명 문구

#### 2. 필터 영역
- 브랜드 필터 (Sony, Canon, Nikon, Fujifilm 등)
- 가격대 필터 (슬라이더 또는 범위 버튼)
- 용도 필터 (풍경, 인물, 스냅 등) - 선택사항

#### 3. 카메라 바디 섹션
- 카드 형태로 카메라 바디 목록 표시
- 각 카드: 카메라 이미지(placeholder), 모델명, 브랜드, 가격, 주요 스펙
- 클릭하면 "선택됨" 상태로 변경

#### 4. 렌즈 섹션
- 카드 형태로 렌즈 목록 표시
- **선택한 바디의 마운트와 호환되는 렌즈만 표시** (핵심 기능)
- 각 카드: 렌즈 이미지(placeholder), 모델명, 초점거리, 조리개, 가격

#### 5. 조합 결과 영역 (하단 또는 사이드바)
- 선택한 바디 + 렌즈 요약
- **합산 가격** 표시
- 구매 링크 버튼 (쿠팡, 네이버 쇼핑 검색 URL로 연결)

#### 6. 푸터
- 출처 및 참고 표시

### Mock 데이터 (JS 배열)
카메라 바디 8~10개, 렌즈 10~15개 정도:
- Sony (A7 IV, A7C II, A6700 등)
- Canon (R6 Mark II, R50 등)
- Nikon (Z6 III, Zf 등)
- Fujifilm (X-T5, X-S20 등)

각 항목에 포함할 정보:
```
바디: brand, model, mount, sensor, price, megapixel, weight, imageUrl
렌즈: brand, model, mount, focalLength, aperture, price, weight, imageUrl
```

### 핵심 JS 기능
1. **필터링**: 브랜드/가격 기준으로 카드 표시/숨김
2. **마운트 호환 체크**: 바디 선택 시 호환 렌즈만 표시
3. **조합 계산**: 선택한 바디+렌즈의 합산 가격
4. **구매 링크 생성**: 네이버 쇼핑 검색 URL 동적 생성
5. **반응형**: 모바일에서도 카드가 잘 보이도록

### 디자인
- 다크 테마 (카메라 사이트 느낌)
- 카드에 호버 효과
- 선택된 카드 하이라이트
- 부드러운 CSS 트랜지션

## Vibe Coding 반복 개선 과정 (PPT용)

| 단계 | 프롬프트 예시 | 결과 변화 |
|------|-------------|---------|
| V1 | "카메라 바디와 렌즈 목록을 보여주는 웹페이지 만들어줘" | 기본 리스트 나열 |
| V2 | "카드 레이아웃으로 바꾸고 스펙 정보를 표시해줘" | CSS Grid 카드 |
| V3 | "브랜드별 필터 기능 추가해줘" | JS 필터링 |
| V4 | "바디 선택 시 호환 렌즈만 보이고 합산 가격 표시해줘" | 핵심 조합 기능 |
| V5 | "구매 링크 버튼과 다크 테마 적용해줘" | 완성도 향상 |
| V6 | "반응형 + 애니메이션 다듬어줘" | 최종 폴리싱 |

각 단계별 스크린샷을 캡처해서 PPT에 포함.
의도적으로 1회 이상 "AI가 잘못한 부분 → 수정 요청" 과정 포함 (과제 요구사항).

## 검증 방법
1. `index.html` 파일을 브라우저에서 직접 열어서 동작 확인
2. 필터 클릭 → 카드 표시/숨김 정상 동작
3. 바디 선택 → 호환 렌즈 필터링 정상 동작
4. 바디+렌즈 선택 → 합산 가격 표시 정상 동작
5. 구매 링크 클릭 → 네이버 쇼핑 검색 페이지로 이동
6. 모바일 화면 크기에서 레이아웃 깨지지 않음
